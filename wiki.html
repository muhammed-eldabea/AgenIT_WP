Project & session

/init [name] [--profile X]

Initialise or repair the AgenIT config in the current directory. Auto-detects a profile from repo markers (package.json → web, CMakeLists.txt+C → embedded, DBC/AUTOSAR → automotive).

  • --profile generic|web|embedded|automotive|design — force a profile.
  • --name <n> — explicit project name (otherwise the directory).
  • --no-agents — skip AGENTS.md seeding.
  • --no-skills — skip .gemini/skills/ seeding.
  • --no-verify — skip the post-init Gemini-CLI sanity check.
agenit init --name my-app --profile web
/unflow

Remove AgenIT-managed config (.agenit/, .gemini/, .flow/, AGENTS.md) from the project. Your project files are untouched. Restores a vanilla Gemini CLI.

agenit /unflow
/project <name>

Switch or create the active project context. Project-scoped memory lives under memory/projects/<name>/.

/project firmware-v2
/yolo on | off | auto_edit

Toggle approval mode for tool calls. on = fully autonomous; off = ask before every tool; auto_edit = autonomous for edits, ask for shell/network.

/yolo auto_edit
/jobs [<id>]

List, inspect, or stop background jobs (long-running tool calls, soul-keeper, overnight loops, goal runners). With no id, lists all; with an id, prints status; stop <id> kills it.

/jobs · /jobs stop GOAL-…-8a11
/configure [hardware]

Detect and configure attached hardware (JLink probes, Saleae logic analyzers, virtual CAN channels). Validates env vars and writes detected serial numbers into config/flow.toml.

/configure hardware
/feature <description>

Start a feature workflow — branch off, scaffold a tasks file, prime the squad. Shorter than running speckit + run by hand for small features.

/feature add OTP login
/help · /clear · /exit

Standard. /help lists every command with one-liners; /clear wipes the scrollback (memory survives); /exit quits cleanly.

/undo [list | <commandId>]

Restore file snapshots taken before every write. With no args, undoes the most recent write. list shows the last 10 snapshots; passing a commandId restores that specific one.

  • /undo list — show last 10 snapshots.
  • /undo <id> — restore a specific snapshot.

Debug Mode & harnesses NEW

/debug on | off | status | pause | release | replay

Turn every chat / LLM / tool / agent call into a typed Run in a debuggable tree. TUI right-rail widget + Web UI tab. Pause-before gates step through risky tools; tool runs are replayable.

  • /debug on · /debug off — toggle the trace recorder.
  • /debug status — current state + active gates.
  • /debug pause <tool> — pause before a given tool fires.
  • /debug release <runId> — release a paused run.
  • /debug replay <runId> — re-run a captured tool invocation.
  • /debug clear — drop the in-memory trace tree.
/debug on → /debug pause shell → /debug release fa3551
Ctrl+W → Harnesses → Enter (TUI widget — there is no /harness slash command)

One-stop add for Skills / MCP / Agents / Tools. Hit n, pick a kind (1–4), name it, choose empty template (t) or scaffolding-agent generated (a), edit in the in-TUI vim editor (:wq saves), review, then y installs to the canonical path with MCP-JSON atomic merge + AGENTS.md de-dup.

  • Skill → .gemini/skills/<name>/SKILL.md (or .claude/skills/…) — auto-activates on semantic description match.
  • MCP → .mcp.json — merged into the shared file; same-id entries overwrite, others survive.
  • Agent → block appended to AGENTS.md — shared by both backends.
  • Tool → .flow/tools/<name>.py (chmod 0755) — callable from hooks, shells, MCP wrappers.
  • r on an installed harness refines it in place with the agent + your feedback.
  • Drafts persist between sessions in ~/.agenit/harnesses/drafts.json.
Ctrl+W → Harnesses → Enter → n

Full reference: docs/harnesses.md.

Ctrl+W widget navigation mode

Enter a keyboard-driven nav layer over the rail widgets. Each widget can expand into a full-pane interactive view (tasks, hardware probes, agents dashboard, recipes, codedigest, debug). Legacy Ctrl+T / Ctrl+] / Ctrl+G shortcuts preserved.

  • ↑/↓ — move focus between widgets.
  • Enter — expand the focused widget to full pane.
  • Esc — collapse / exit nav mode.
  • Ctrl+W — toggle nav.

Backends & failover NEW

/backend [models <provider> | chain <tier>]

Runtime visibility for the multi-backend layer. With no args, prints a diagnostic table of detected backends, installed binaries, and the currently active provider + model.

  • /backend — detected / installed / active table.
  • /backend models <provider> — list available models for gemini-cli, claude, claude-cli, openai, ollama.
  • /backend chain <tier> — show the cross-backend failover chain (e.g. pro, flash).
/backend models claude-cli
[backend] provider = "claude-cli"

Spawn the locally installed claude binary as a subprocess backend. Uses your existing Anthropic OAuth (Pro / Max / Team / Enterprise) — no API key required. Distinct from provider = "claude", which goes straight to the Anthropic Messages API with ANTHROPIC_API_KEY.

  • binary — path to the claude binary (default: from PATH).
  • model — e.g. claude-sonnet-4-5.
  • fallbackModel — model to drop to on rate-limit.
  • requestTimeoutSecs — per-request wall-clock cap (default 120s, 0 disables).
  • settingsPath — explicit .claude/settings.json path for hook bindings.
agenit init --target claude | gemini | both

Seed .claude/, .gemini/, or both. Hooks under .flow/hooks/ are shared between backends — one hook, both runtimes.

agenit init --target both
Interactive backend picker

agenit init probes which backends you have installed and asks you to pick when stdin is a TTY. Non-interactive runs (CI) fall back to --target gemini.

Pipeline (V-Model)

/run <task>

Full pipeline: SWE.1 Requirements → SWE.2 Architecture → SWE.4 Code → SWE.5 Testing → SWE.6 Critic → Audit. Each stage activates a different skill. Memory + soul ride along on every stage.

/run add CAN timeout recovery to can_driver.c
/swe1 <task>

Requirements stage — writes EARS user stories with SWR-NNN / REQ-NNN tags into requirements.md.

/swe1 add password reset flow
/swe2 <task>

Architecture stage — HLD + PlantUML diagrams written to architecture.md.

/swe2 design the message bus
/swe4 <task>

Implementation stage — MISRA-compliant C / language-appropriate code with Doxygen and @req tags linking back to SWR-NNN.

/swe5 <task>

Testing stage — Unity / CppUTest / Vitest / Playwright tests covering every requirement. Optional HIL hooks.

/swe6 <task> NEW

Critic stage — swe6-critic stress-tests the output of every other stage for vague verbs, contradictions, and untestable claims. Writes critique.md.

/audit

Traceability + compliance report — requirements covered, orphan tests, missing audits, ASPICE checklist.

/traceability

Show the @req graph linking requirements → code → tests → audit notes. Includes the new REQ-SKILL-NNN skill inventory.

Mission Squad

/squad implement <task>

Deterministic helpers (req-reader · code-scanner · trace · MISRA · codedigest) → swe4-implementer. Helpers run in parallel; the implementer fires with a pre-injected briefing.

/squad implement-agentic <task>

Same shape as implement but every helper is its own Gemini sub-agent loaded with a focused skill. Heavier but more flexible.

/squad debug <task>

/squad review <task>

MISRA + req + code → embedded-misra-review (or language-appropriate reviewer).

/squad test <task>

Req + code + memory → swe5-tester.

/squad requirements <task>

/squad architecture <task>

Req + memory + headers → swe2-architect.

/squad <template>-agentic <task>

Autonomous Goals

/goal <objective> [--force]

Create or replace the active goal. Long-lived Plan → Code → Test → Evaluate loop with budget tracking + auditor. State checkpointed under memory/projects/<name>/goal.json.

  • /goal <objective> — create (fails if one is active).
  • /goal <objective> --force — replace the active goal.
  • /goal (no args) — short status (id, state, turns).
/goal Migrate the entire codebase to TypeScript
/goal status

Verbose dashboard — id, state, turns used / remaining, files touched, tests added, errors, runner status.

/goal log [n]

Audit log — every stage transition, every auditor decision. With n, shows the last n entries.

/goal start [--interactive]

Spawn the autonomous runner in the background. The REPL stays interactive; the runner survives REPL exit.

  • --interactive — prompts for turn budget & cadence at launch, and pauses for review at each per-tick checkpoint (audit #134). NEW
  • Auto-checkpoint cadence is configurable via [goal] checkpoint_every_turns in agenit.toml.
/goal stop

Cancel the runner cleanly. State is preserved; /goal start resumes.

/goal tick

Step a single turn manually. Useful for stepping through a goal interactively.

/goal pause · /goal resume

Flip state mid-run. Runner stays alive; just stops issuing turns.

/goal budget <n>

Extend the turn budget by n. Useful when an auditor decides "Continue" but the cap is reached.

/goal clear

Archive the active goal to goals_archive.md and free the slot for a new one.

Overnight loop NEW

/overnight [flags] -- <shell cmd>

Long-running autonomous loop with per-iteration git commits, rollback on failure, worktree-isolated parallel runs, and a permanent markdown summary. Agent-agnostic. Inspired by gnhf.

  • --max-iter N — cap iterations (default 50).
  • --budget-tokens N — stop once a tick reports tokensSpent ≥ N.
  • --parallel N — run N worktree-isolated copies in parallel.
  • --no-commit — dry-run; no commits, no rollback.
  • --allow-dirty — skip the clean-tree precondition.
  • --summary <path> — where to write the permanent run summary.
/overnight --parallel 2 --max-iter 30 -- gemini "fix the failing tests"

Agile · OPA · QA · PMO NEW

/agile [enable | disable | status | digest | burndown | sprint start | sprint status | next-sprint]

Manage the Agile persona layer — enable / disable Scrum ceremonies and the persona crew. The ceremony worker runs in the background and fires standup / planning / review / retro on cadence; OPA and QA gates plug in automatically.

  • /agile enable · /agile disable — toggle the layer and the ceremony worker.
  • /agile status — sprint state, cadence, token budget, ceremony worker status.
  • /agile sprint start · /agile sprint status · /agile sprint approve-scope — manual sprint control.
  • /agile next-sprint — close the active sprint, archive metrics, open the next.
  • /agile digest — per-project digest: ceremonies fired, blockers, velocity. NEW
  • /agile burndown — ASCII burndown chart for the active sprint. NEW
/agile enable && /agile burndown
/opa [status | enable <cat> | disable <cat> | upload <path> | reset <cat> | init]

Manage Organizational Process Assets — upload policies, templates, and lessons-learned. OPA assets are auto-injected into the prompt for matching ceremonies and stages.

  • /opa status — categories + counts.
  • /opa upload <path> — ingest a policy / template / lesson file.
  • /opa enable <cat> · /opa disable <cat> — toggle a category.
  • /opa reset <cat> — clear back to defaults.
  • /opa init — seed the store with the bundled defaults.
/qa [--goal <id>] [--spec <path>] [--fix]

Verified deliverables + scope validation gate — writes qa-report.md with LLM-evaluated acceptance criteria, Definition-of-Done checks, and PMO QA failure detection.

  • --goal <id> — scope QA to a specific active goal.
  • --spec <path> — point at a non-default spec file.
  • --fix — attempt auto-fix of QA gate failures.
/qa --goal G-0042
/pmo [status [--project <name>] | report | forecast [--backlog <N>]]

Portfolio governance dashboard — RAG status across all active projects, long-form PMO report, and velocity-based backlog completion forecast.

  • /pmo status — RAG (red / amber / green) across every project the orchestrator has touched.
  • /pmo report — long-form report (budget, velocity, blockers, ceremonies, OPA hits).
  • /pmo forecast --backlog 25 — predict completion date for the next N backlog items.

/migrate gemini-plan list [--cache <dir>] | copy <path|--latest> [--slug <name>] [--force] NEW

Recover raw Gemini CLI plan-mode artefacts so they can run through the AgenIT pipeline. Lists every plan-mode artefact found on disk and copies the chosen one into .specify/specs/<slug>/ for /speckit + /run.

Critic & PR review NEW

/diff-walk [--base <ref>] [--inventory-only]

Walk git diff hunks; run swe6-critic on each; print a severity-sorted report.

  • --base <ref> — compare against a base ref (default: main).

Critic per hunk + apply deterministic patches (lint / format / import-order / trailing-whitespace) with backup branch + telemetry.

  • <pr#> — GitHub PR number (uses gh).
  • --confirm — apply patches non-interactively.
  • --dry-run — print the patch plan, do not write.
/pr-autofix 1421 --confirm
/pr-metrics [--since <iso>] [--json]

AI-vs-human cohort report from events.jsonl — writes / cost / tool calls / autofix activity.

  • --since 2026-01-01 — filter window.
  • --json — machine-readable output for dashboards.

Code search & call graph

/codedigest index <path>

Build the BM25 index for the given path. Stored as JSON under memory/projects/<name>/codedigest_<name>.json — no native deps, no embeddings download.

/codedigest search <query> [--lang X] [--graph]

Top results, ranked. --graph attaches each hit's direct callers + callees (DKB-style retrieval).

  • --lang c|ts|py|... — filter by language.
  • --graph — auto-inject callers + callees (needs graph --use-lsp first).
  • --limit N — change the top-N (default 5).
/codedigest graph --use-lsp [--export svg|dot|html] NEW

Build the LSP-derived call graph (clangd / pyright / rust-analyzer; daemon-mode reuse). Render to Graphviz / SVG / interactive HTML.

/codedigest graph --use-lsp --export html
/codedigest status

Show index stats — file count, chunk count, last-built timestamp.

/codedigest clear

Drop the index.

CLI flags & environment
